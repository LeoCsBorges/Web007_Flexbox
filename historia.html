História do Flexbox

O módulo de configuração Flexbox (Flexible Box, caixa flexível) tem como objetivo prover uma maneira mais eficiente de dispor, alinhar e distribuir espaços entre itens em um recipiente (container), mesmo quando seus tamanhos são desconhecidos e/ou dinâmicos (por isso, a palavra "flex").

A ideia principal por detrás do flex layout é dar ao container a habilidade de alterar a largura e altura de seus itens para melhor preencher o espaço disponível (principalmente para acomodar em todos os tipos e tamanhos de telas de diferentes dispositivos). Um container flex expande os itens para preencher o espaço livre disponível ou os encolhe para previnir que extravazem (overflow).

Mais importante, o flexbox layout é direcionamento-agnóstico em oposição aos layouts comuns (block que é baseado verticalmente e inline que é horizontalmente). Enquanto esses funcionam bem para páginas, lhes falta flexibilidade para suportar grandes e complexas aplicações (especialmente quando é necessário mudar, redimensionar, esticar, contrair, etc.).

▹Nota: o Flexbox layout é mais apropriado para componentes de uma aplicação, e layouts de pequena escala, enquanto o Grid layout é destinado para layouts de grande escala.
